이미지를 분석한 결과, 기획서는 "추천 장비 기능 개선"에 관한 내용으로 보입니다. 캐릭터 장착 화면과 추천 장비 리스트의 UI 개선(기본 → 개선)을 다루고 있습니다. 이미지 해상도가 낮아 일부 텍스트는 정확히 판독되지 않지만, 시각적으로 파악 가능한 UI 요소와 일반적인 추천 장비 기능 플로우를 기반으로 TC 작성 계획을 수립하겠습니다.

### 기능 목록

1. **캐릭터 장착 화면 진입 시 추천 장비 표시**
   - 캐릭터 정보/스탯 영역 출력
   - 캐릭터 장착 슬롯(현재 장착 장비) 출력
   - 추천 장비 영역(추천 받은 장비 리스트) 출력

2. **추천 장비 표시 규칙**
   - 보유 장비 중 추천 조건에 부합하는 장비 출력
   - 추천 기준(콘텐츠별/캐릭터별)에 따른 장비 노출
   - 추천 가능한 장비가 없는 경우 처리
   - 1차/2차 추천 단계 구분 표시

3. **추천 장비 리스트 UI**
   - 추천 장비 슬롯 (다수의 장비 노출)
   - 장비 아이콘/등급/레벨 표시
   - 장비 선택 시 하이라이트 처리
   - 스크롤/페이지 처리 (장비 수가 많을 경우)

4. **추천 장비 선택 및 적용**
   - 추천 장비 터치 시 상세 정보 표시
   - 장비 선택 시 캐릭터에 미리보기 적용
   - 적용 버튼 터치 시 장비 변경 확정
   - 변경 후 스탯 변화 반영

5. **장비 비교 기능**
   - 현재 장착 장비 vs 추천 장비 스탯 비교
   - 증가/감소 수치 색상 구분 표시

6. **개선 전/후 UI 차이**
   - 기존 UI 대비 추천 장비 가시성 강화
   - 추천 장비 영역의 위치/크기 변경

7. **알림 시스템**
   - 추천 가능한 장비 보유 시 레드닷 출력
   - 추천 장비 확인 후 레드닷 제거

8. **뒤로가기 처리**
   - 추천 장비 화면에서 뒤로가기 동작

9. **예외 처리**
   - 추천 가능한 장비 없음
   - 모든 추천 장비가 이미 장착된 경우

### 중점 확인 사항

1. **이미지 해상도 한계**: 기획서 이미지가 저해상도로 정확한 수치(추천 장비 개수, 등급 조건, 단계별 기준치 등)가 판독되지 않음. 시각적으로 확인 가능한 UI 동작 위주로 작성하며, 추측이 필요한 수치는 TC에 포함하지 않는다.

2. **기존(기본) vs 개선 UI 차이**: 기획서 의도가 "기능 개선"이므로, 개선된 UI의 동작(추천 장비 영역 표시, 노출 위치 변경 등)을 중점적으로 검증한다. 기존 UI는 별도 TC로 작성하지 않는다.

3. **추천 규칙 검증**: 추천 장비가 어떤 기준(콘텐츠별, 캐릭터별, 등급별)으로 노출되는지 정확한 텍스트 판독이 어려우므로, "추천 조건에 부합하는 장비가 출력된다" 수준의 일반화된 표현으로 작성한다.

4. **장비 변경 확정 플로우**: 추천 장비를 선택→적용 버튼→확정으로 이어지는 핵심 플로우는 데이터 변경(장비 교체)이 발생하므로 Critical/Major로 분류하여 빠짐없이 검증한다.

5. **스탯 비교 표시**: 추천 장비 선택 시 현재 장비 대비 증가/감소 스탯이 색상으로 구분되어 표시되는 부분을 별도 sub3로 분리하여 검증한다.

6. **알림 시스템**: 추천 가능 장비 보유 시 레드닷 출력 및 확인 후 제거 조건을 반드시 포함한다.

7. **예외 케이스 배치**: "추천 장비 없음", "모든 추천 장비가 이미 장착됨" 등 예외 상황은 각 sub2 카테고리의 가장 마지막에 배치한다.

8. **인게임 제외**: 추천 장비 적용 후 인게임 진입 관련 TC는 작성하지 않는다.
