### 기능 목록

**[1] 모드리스트 화면**
- 자동추천 토글 (ON/OFF)
- 다이어리(Today) 진입 버튼, 닫기(X) 버튼
- 모드 카드 출력 (일일도전, 불멸하는 용의 전설, 트로피 쟁기, 맵 메이커, 챔피언스 리그)
- 각 모드 카드 상단 레드닷/배지 표시
- 모드 시즌 종료 타이머 표시
- 맵 메이커 카드 선택 시 모드 선택 화면 진입

**[2] 맵메이커 모드 선택 화면**
- "만들기" 버튼, "달리기" 버튼 출력 (달리기는 다음 TF — 본 TC 제외)
- 만들기 버튼 터치 시 만들기 모드 메인 진입
- 뒤로가기 동작

**[3] 만들기 모드 메인 (나의 맵 목록)**
- 상단 재화 표시(코인, 크리스탈)
- 홈 버튼
- 등록된 맵 슬롯 (맵 코드 상단 표시, 평가수 하단 표시)
- 미등록 슬롯 (등록된 이력 있을 경우 평가수 표시)
- 빈 슬롯 (+ 버튼)
- 슬롯 패키지 추가 규칙
- 등록 해제 시 평가수 초기화 여부

**[4] 맵 생성**
- 맵 이름 입력 필드 (필수)
- 맵 설명 입력 필드 (필수)
- 맵 배경 선택 (필수, 기본 첫번째 선택)
- 맵 배경 자세히 보기 팝업
- 필수 정보 입력 전 생성하기 버튼 비활성화
- 필수 정보 입력 시 생성하기 버튼 활성화
- 샘플 사용하기 선택 시 Sample 1(EASY)/Sample 2(NORMAL)/Sample 3(HARD) 출력
- 샘플 사용하기 버튼 → 선택한 샘플 맵으로 표시
- 닫기/뒤로가기

**[5] 선택 맵 메인 화면**
- 맵 삭제 버튼
- 도움말 버튼
- 닫기(X) 버튼
- 맵 이름 출력
- 플레이어 설정(카메라 거리, 이동 속도)
- 추가 규칙(규칙 없음/규칙 있음)
- 등록 상태 표시(맵 코드 출력, 좋아요/싫어요 카운트)
- 맵 설정 버튼
- 등록 테스트 버튼
- 편집하기 버튼

**[6] 등록 테스트 (인게임)**
- 로딩 화면 진입
- 시작 멘트 출력
- 플레이 화면(Jump/Slide 버튼, 진행도, 코인, 미션 카운터)
- 완주(미션 성공 체크 후 연출)
- 중간 미션 실패 시 처리
- 실패 처리

**[7] 결과 화면**
- 성공 결과 (SUCCESS, 점수, 코인, 미션 체크 리스트, 등록 버튼)
- 실패 결과 (FAIL, 점수, 코인, 미션 체크 리스트, 확인 시 메인 이동)
- 달리기 모드 결과 (다시하기 / 닫기 버튼)

### 중점 확인 사항

1. **달리기 모드 제외 처리**
   - 기획서 명시: "달리기 모드는 다음 TF 때" → 달리기 버튼은 출력 TC만 작성하고, 진입 후 기능 TC는 작성하지 않음.

2. **맵 생성 필수 입력 검증**
   - 맵 이름/맵 설명/맵 배경 모두 필수 항목. 미입력 시 생성하기 버튼 비활성화, 입력 완료 시 활성화 → Critical/Major로 분리.

3. **샘플 사용 분기 처리**
   - 빈 슬롯 진입 → 일반 생성 vs 샘플 사용하기 분기. 샘플 선택 후 사용하기 클릭 시 선택한 샘플로 맵 정보 자동 채움.

4. **등록 상태 UI 차이**
   - 등록된 맵: 상단 맵 코드(예: ABCDE1234) + 좋아요/싫어요 카운트 표시
   - 등록 해제된 맵(이력 있음): 평가수만 표시, 초기화 여부는 추측 금지 — 기획서 상 "초기화되어야 하나?"로 미확정이므로 해당 부분은 TC에서 제외 또는 "평가수 표시" 수준으로만 작성.

5. **인게임 영역 TC 제외**
   - 등록 테스트 진입 후 로딩→플레이→종료까지의 흐름 중, "Play 버튼 터치 → 로딩 진입" 수준까지만 작성하고 실제 인게임 동작은 제외하는 것이 원칙이나, 기획서에 등록 테스트가 핵심 플로우로 포함되어 있으므로 **로딩 화면 진입 / 결과 화면 진입** 수준까지의 화면 전환 TC는 포함. 인게임 조작(점프/슬라이드 실제 작동, 캐릭터 이동 등)은 TC 제외.

6. **결과 화면 분기**
   - 성공(SUCCESS) / 실패(FAIL) / 달리기 단독 진행 시 3가지 분기 별 UI 및 버튼 구성이 다름 — 각각 분리하여 작성.

7. **편집하기 진입**
   - 편집하기 버튼은 별도 편집 화면으로 진입하나, 본 기획서 범위에서는 해당 화면 상세 기능이 명시되지 않음 → "편집 화면으로 진입된다" 수준의 화면 전환 TC만 작성, 편집 화면 내부 기능 TC는 추측하지 않음.

8. **슬롯 확장 규칙**
   - "슬롯은 패키지처럼 늘려가기"로만 명시되어 있고 구체 스펙 미정 → 슬롯 표시 TC만 작성하고 확장 기능 TC는 추측하지 않음.

9. **카테고리 마지막 배치 규칙**
   - 화면 이동 TC(예: 만들기 버튼 → 만들기 모드 메인 이동)는 각 카테고리 마지막에 배치.
   - 예외(맵 삭제 확인 팝업의 취소 동작, 닫기 동작 등)도 카테고리 끝에 배치.
