기획서 이미지를 분석했습니다. "반죽 게임즈 테스트 센터" 이벤트의 메인 로비 화면 1장이 제공되었습니다. 단일 화면 기반으로 TC 설계 계획을 수립하겠습니다.

### 기능 목록

기획서 이미지에서 파악한 기능은 다음과 같습니다.

1. **이벤트 진입 / 좌측 이벤트 리스트 사이드바**
   - 이벤트 리스트 출력 (출석부, 쿠키 출시 이벤트, 네온데비쉬맛 쿠키 픽업 이벤트, 플레이 이벤트, 반죽 게임즈 테스트 센터, 어드벤처 모드 부스팅 이벤트, 신비한 연금술 공방, 어비스 탐색 보고서)
   - "이벤트" 헤더 타이틀 출력
   - 현재 선택된 이벤트 항목 강조 표시 (반죽 게임즈 테스트 센터 - 노란색 하이라이트)
   - 이벤트 항목 터치 시 화면 전환

2. **상단 영역**
   - 이벤트 잔여 시간 타이머 출력 (예: 22일 16시간)
   - 타이머 카운트다운 동작

3. **반죽 게임즈 테스트 센터 메인 화면 UI**
   - 이벤트 타이틀 이미지 출력 ("반죽 게임즈 테스트 센터")
   - 중앙 메인 캐릭터(고양이 형태 쿠키, 안경 착용) 출력
   - 캐릭터 대사 말풍선 출력 ("랭킹에 도전할 거야? 아옹- 과연 내 신기록을 이길 수 있으려나?")
   - 우측 TV/모니터 프리뷰 영역 출력
   - 배경(크리스탈/장식 요소) 출력

4. **중앙 메뉴 버튼**
   - 튜토리얼 버튼 (돋보기 아이콘) 출력 및 터치 동작
   - 랭킹 버튼 (트로피 아이콘) 출력 및 터치 동작
   - 미션 버튼 (문서 아이콘) 출력 및 터치 동작

5. **START 버튼**
   - START 버튼 출력 (우측 하단, 노란색)
   - START 버튼 터치 시 콘텐츠 진입

6. **예외 / 부가 동작**
   - 이벤트 기간 종료 시 처리
   - 디바이스 뒤로가기 동작
   - 버튼 터치 효과음, 화면 전환 연출

### TC 건수 배분

제공된 자료가 단일 스크린샷이고 UI 플로우 문서가 없는 관계로, 추측 없이 시각적으로 확인 가능한 요소 위주로 작성합니다. 무리하게 100건을 채우지 않고 커버리지 기준으로 작성합니다.

- **전체 예상 TC**: 약 30~38건
- **Priority 분배**:
  - Critical: 약 4건 (12%) — START 버튼을 통한 콘텐츠 진입, 핵심 메뉴(랭킹/튜토리얼/미션) 진입 차단 관련
  - Major: 약 16건 (45%) — 이벤트 리스트 출력/전환, 메인 UI 요소 출력, 타이머 동작, 버튼 활성 상태 등
  - Normal: 약 14건 (40%) — 효과음, 애니메이션 연출, 대사 말풍선, 배경 연출, 프리뷰 영역 등 부가 UI

| 카테고리 | 예상 TC 수 | Critical | Major | Normal |
|---|---|---|---|---|
| 이벤트 리스트 | 6~7건 | 0 | 4 | 2 |
| 상단 영역(타이머) | 2~3건 | 0 | 2 | 1 |
| 메인 화면 UI | 6~7건 | 0 | 3 | 4 |
| 메뉴 버튼 영역(튜토리얼/랭킹/미션) | 9~10건 | 3 | 4 | 3 |

| 예외 처리 | 2~3건 | 0 | 2 | 1 |

### 중점 확인 사항

1. **이미지 단일 자료 기반 작성**: UI 플로우 문서와 텍스트 기획서가 제공되지 않아, 스크린샷에서 시각적으로 확인 가능한 요소만 TC화하고, 튜토리얼/랭킹/미션 진입 후 세부 화면은 추측 작성하지 않습니다(각 버튼 터치 시 해당 화면으로 전환되는 동작까지만 작성).

2. **이벤트 진입 플로우의 핵심 기능 식별**: START 버튼 터치 시 콘텐츠 진입은 이벤트의 핵심 진입점이므로 Critical로 분류합니다. 튜토리얼/랭킹/미션 버튼은 각각 별도 콘텐츠로 진입하는 핵심 메뉴이므로 진입 동작 자체를 Critical로 다룹니다.

3. **이벤트 리스트 사이드바 통합 처리**: 좌측 이벤트 리스트는 "반죽 게임즈 테스트 센터"가 위치한 이벤트 화면 진입 경로이므로 같은 main "이벤트" 카테고리 내 sub1으로 묶어 작성합니다. 다른 이벤트(출석부, 쿠키 출시 이벤트 등)는 별도 기획서가 아닌 리스트 표시 항목으로만 다룹니다.

4. **카테고리 통합 원칙 적용**: 튜토리얼/랭킹/미션 진입 화면을 별도 main으로 분리하지 않고, "반죽 게임즈 테스트 센터" sub1 하위의 메뉴 버튼 sub2 내 진입 동작(sub3)으로 묶어 처리합니다.

5. **화면 이동 케이스 / 예외 케이스 배치**: 각 sub2 카테고리 끝에 화면 전환 TC를 배치하고, 이벤트 기간 종료, 디바이스 뒤로가기 등 비정상 플로우는 카테고리 가장 마지막에 배치합니다.

6. **시각 정보 한계 명시**: 프리뷰 영역(녹색 테두리 TV)이 무엇을 표시하는지, 캐릭터 대사가 동적으로 변경되는지, 타이머 형식이 정확한지 등은 이미지만으로 단정할 수 없어 "출력된다" 수준으로 작성하고 동작 세부 사항은 추측하지 않습니다.
