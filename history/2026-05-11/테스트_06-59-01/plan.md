# TC 설계 계획

## 기능 목록

### 1. 레이드 모드 진입 화면 (햄버거 → 레이드 모드)
- 화면 진입 (로비 → 햄버거 → 레이드 모드 전환)
- 뒤로가기 버튼
- 상시/이벤트 토글 탭
- 싱글/멀티 모드 전환 토글 (Default: 싱글)
- 보스 진입 카드 리스트 (불타는 곳, 성장 던전, 탑의 균열, 어비스 던전 등)
- 보스 카드 정보 표시 (보스 명/이름, 보상, 도입 정보, 슈퍼에픽 세트 정보 최대 3종)
- 잠금 상태 아이콘 표시 및 잠금 보스 터치 시 안내 팝업
- 보스 카드 선택 시 레이드 로비로 전환

### 2. 레이드 모드 로비 (싱글 중심)
- 싱글/멀티 토글 버튼 (Default: 싱글)
- 보스 정보 영역 (보스 이름, 초상화, 배경 이미지)
- 보스 정보 팝업 출력
- 난이도 변경 버튼 / 팝업 출력 / 재클릭 시 HIDE
- 보스·난이도 변경 팝업 (보스 변경, 단계 변경, 10단계)
- 입장 전투력 표시 / 부족 시 비활성화 + Toast (equipment.dungeon.not.host)
- 이전 난이도 미클리어 시 비활성화 + Toast (single_boss_challenge.locked)
- 추천 속성 표시
- 적용 조건 표시
- 빠른 난이도 변경 버튼 + 위 두 Toast 케이스
- 보상 정보 영역 / 상세 보상 팝업
- 보상 상세 팝업 보스 변경 (좌우 버튼)
- 난이도별 보상 정보 (기본 보상, 클리어 연관 보상)
- Play 버튼
- 배수 설정 (기본 최대 3장, 패스 구매 시 최대 5장)
- 티켓 입장 변경 (5장→2장)
- 멀티 모드 전환 (매칭/파티 영역)
- 난이도 변경 안내 팝업 (보스 진입 팝업) / 닫기 버튼

### 3. 레이드 인게임 (인게임 자체는 제외하지만 진입/클리어 전환 및 보상 표시는 포함 - 단 인게임 제외 규칙 검토 필요)
실제로 "Play 버튼 누른 후 인게임"은 제외 규칙에 해당. 하지만 클리어 화면 보상 표시 / 나가기 / 드랍 연출 등은 결과 UI라 포함 검토.
규칙: "인게임 관련 케이스는 작성하지 않는다" → 인게임 전투, 컨트롤(점프/공격), 보스 체력, 제한시간 등은 제외
하지만 "획득 아이템 정보 표시", "클리어 화면 보상", "나가기 버튼", "드랍 연출 제거(코코방울 등)"는 결과 UI/시스템이므로 포함
- 드랍 연출 (3초간 표시)
- 코코방울 연출/축복 포인트/랜덤 상자/트로피 포인트 제거 확인
- 획득 아이템 표시 (장비, 잠재력 항아리, 설계도, 조각, 연금 항아리)
- 아이템 클릭 시 툴팁
- 12종 이상 시 스크롤
- 다시까기 기능 제거
- 나가기 버튼

### 4. 챌린지 레이드 모드 로비
- 모드 전환
- 보스 진열/PLAY 버튼
- 강무어보 팝업 (점수, 단계 진행)
- 챕터 보상 미션
- 코코방울/트로피/무지개 상자 제거 확인
- 4인→2인 멀티 변경
- 일반 레이드 랭킹 진입 버튼 제거
- 클리어 화면 트로피/축복/무지개 영역 제거

### 5. 제작소 화면
- 로비 → 햄버거 → 제작소 전환
- 뒤로가기
- 메뉴 정렬 (장비 제작 → 장비 연금 → 아티팩트 제작 → 아이템 제작)
- 장비 연금 메뉴 버튼
- 장비 연금 진입 (기능 자체는 제외 항목이지만 진입 전환만 포함)

### 6. 레이드 모드 탑레벨 화면
- 탑레벨 메뉴 트리 (성장 던전, 레이드 모드, 탑의 균열, 어비스 던전)
- 레이드 모드 선택 상태
- 보스 리스트 표시
- 진입 버튼(→) 클릭 시 해당 보스 로비 진입

## 중점 확인 사항

1. **제외 항목 엄수**: 변경 플로우 / 연금술 이벤트 삭제 / 장비 연금 상시 시스템 편입 + 연금 항아리 드랍 / 탑레벨 6종 보스 정보 표시 / 데이터 테이블 / 이펙트 요청 리스트 → TC 미작성. 단, 장비 연금 메뉴 "진입 전환"은 제작소 화면 카테고리에 포함.

2. **화면 전환 누락 방지**: 각 화면 마지막에 다음 화면으로의 전환 케이스 배치 (로비→레이드 진입, 진입→로비, 로비→인게임, 인게임→클리어→로비, 챌린지 모드 전환, 로비→제작소, 제작소→장비 연금, 탑레벨→보스 로비).

3. **싱글/멀티 토글 Default 검증**: Default가 싱글이어야 한다는 규칙을 진입 화면, 로비 양쪽에서 검증.

4. **Toast 메시지 정확 표기**: `equipment.dungeon.not.host`(전투력 부족), `single_boss_challenge.locked`(이전 난이도 미클리어) — 각 발생 지점별로 분리하여 작성.

5. **수치 검증**: 티켓 입장 5장→2장 / 기본 3장 패스 5장 / 10단계 / 드랍 연출 3초 / 12종 표시 / 멀티 4인→2인.

6. **제거 기능 검증**: 코코방울 연출·축복·축복 포인트, 트로피 시스템·상자·포인트, 무지개 상자, 랜덤 상자, 다시까기, 일반 레이드 랭킹 — 잔존 여부 확인 케이스를 발생 지점별(인게임 결과/챌린지 로비/클리어 화면)로 작성.

7. **단일 기대결과 원칙**: 보스 카드 정보(이름/보상/세트), 보상 팝업 정보(기본 보상/연관 보상), 단계 버튼 정보(단계 표시/입장 전투력) 등은 각각 별도 TC로 분리.

8. **인게임 제외 규칙**: 인게임 전투(점프/공격 컨트롤, 보스 체력, 제한시간 03:12 등)는 작성 제외. 단 클리어 결과 화면의 보상 표시·드랍 연출·나가기 버튼·제거 기능 검증은 시스템 결과 UI이므로 작성.

9. **잠금 상태 케이스**: 보스 잠금 / 난이도 잠금 / 자물쇠 표시 / 잠금 안내 팝업 출력을 별도 분리.

10. **예외 케이스 배치**: 팝업 재클릭 시 닫힘 / 입장 티켓 부족 시 동작 / 멀티 매칭 실패 등은 해당 카테고리 마지막에 배치.

위 계획대로 진행해도 될지 확인 부탁드리며, 승인 시 2단계에서 JSON 형식 TC를 작성하겠습니다.
