기획서를 검토한 결과, 입력 자료는 게임 기획서가 아닌 나무위키 "쌀먹" 용어 설명 문서로 확인됩니다. 게임 기능/UI 플로우/버튼 동작 등 테스트 가능한 기능 명세가 없으나, 작업을 진행하라는 지침에 따라 페이지 자체의 UI 요소(헤더, 분류 태그, 펼치기/접기 토글, 목차 네비게이션, 본문/하이퍼링크 등)를 테스트 대상으로 간주하여 TC 설계 계획을 수립하겠습니다.

### 기능 목록

기획서(나무위키 페이지)에서 파악한 테스트 가능 기능은 다음과 같습니다.

1. **페이지 진입 / 헤더 영역**
   - 나무위키 로고 출력
   - 페이지 제목("쌀먹") 출력
   - 최근 수정 시각 출력
2. **분류 태그 영역**
   - 4종 분류 태그 출력(게임 용어 / XX충 / 축약어 / 리워드 애플리케이션)
   - 각 분류 태그 터치 시 해당 분류 페이지로 이동
3. **비디오 게임 용어 배너**
   - 배너 출력
   - 펼치기/접기 토글 동작
4. **목차(목차 박스)**
   - 목차 출력(5개 대분류, 하위 항목 포함)
   - 각 항목 터치 시 본문 해당 섹션으로 이동(앵커 스크롤)
5. **본문 영역**
   - 1. 개요 / 2. 유사 개념(2.1 의미 확장) / 3. 상세 / 4. 사례(4.1~4.9, 4.3.1 Mapleland, 4.9.1 유형) / 5. 관련 문서 본문 출력
   - 본문 내 내부 하이퍼링크(파란색 텍스트) 터치 시 해당 문서로 이동
   - 본문 내 외부 링크(#관련기사 등) 터치 시 외부 웹페이지 이동
6. **스크롤 / 페이지 동작**
   - 세로 스크롤 동작
   - 디바이스 뒤로가기 동작
7. **예외 처리**
   - 네트워크 단절 시 페이지 로딩 처리
   - 앱 종료/재진입 시 마지막 스크롤 위치 처리

### TC 건수 배분

전체 목표 약 45~55건(기획서 규모가 작아 100건 무리 충당 지양). Priority 분배 다음과 같습니다.

| 카테고리 | 예상 건수 | Critical | Major | Normal |
|---|---|---|---|---|
| 페이지 진입 | 5건 | 1 | 2 | 2 |
| 분류 태그 | 3건 | 0 | 2 | 1 |
| 비디오 게임 용어 배너 | 4건 | 0 | 2 | 2 |
| 목차 | 10건 | 1 | 6 | 3 |
| 본문 | 17건 | 0 | 2 | 15 |
| 본문 내 링크 | 4건 | 1 | 3 | 0 |
| 페이지 동작 | 5건 | 1 | 2 | 2 |
| **합계** | **약 48건** | **약 4건(8%)** | **약 19건(40%)** | **약 25건(52%)** |

- Critical: 페이지 첫 로딩, 목차 네비게이션 핵심 동작, 외부/내부 링크 진입 등 진입 차단 위험 케이스
- Major: 토글 동작, 분류 태그 터치, 스크롤/뒤로가기 등 동적 동작
- Normal: 단순 정적 출력(제목, 수정 시각, 본문 텍스트 출력 등)

### 중점 확인 사항

1. **자료 한계 명시**: 입력 자료가 게임 기획서가 아닌 위키 문서이므로, 게임 내 기능(전투/육성/상점/이벤트 등)에 대한 TC는 작성 불가. 페이지(문서 뷰어) UI 동작 위주로만 작성.
2. **UI 요소 위치 정확성**: 헤더(상단), 분류 태그(헤더 하단), 배너(분류 하단), 목차 박스(배너 하단), 본문(목차 하단) 순서대로 위치 명시.
3. **목차 항목별 분리**: 5개 대분류 + 사례 하위 항목(11개)에 대한 앵커 이동 케이스를 sub3로 분리하여 검증 누락 방지.
4. **링크 종류 구분**: 내부 위키 문서로 이동하는 파란색 하이퍼링크와, 외부 페이지로 이동하는 #관련기사 류 외부 링크를 별도 sub2로 분리.
5. **펼치기/접기 토글**: 동일 UI지만 펼치기 동작과 접기 동작은 각각 다른 결과 → sub3로 분리.
6. **안내 문구 일반화**: 본문 내 구체적 문장을 그대로 인용하지 않고 "해당 섹션 본문이 출력된다" 형태로 일반화.
7. **인게임/비기능 제외 규칙 준수**: 폰트/해상도/디자인 검증 케이스 제외, 게임 인게임 진입 후 로직 미작성.
8. **추측 금지**: 위키 문서에 명시되지 않은 검색창/광고/계정 메뉴 등 일반적 위키 UI는 자료에서 확인되지 않으면 TC로 작성하지 않음.
