### 기능 목록

**1. 모드리스트 (맵 메이커 진입점)**
- 자동추천 토글, 다이어리 버튼, Today 버튼, 닫기(X) 버튼
- 모드 카드 (NORMAL/HARD 배지, 타이틀, 설명, 남은 시간 표시)
- 맵 메이커 카드 선택

**2. 맵메이커 모드 선택 화면**
- 만들기 버튼 (이번 TF 적용 대상)
- 달리기 버튼 (다음 TF로 예정 → TC 제외)

**3. 만들기 모드 메인 - 나의 맵 목록**
- 상단 재화 표시(1000, 50000), 홈 버튼, 정렬 토글
- 나의 맵 슬롯 목록 (등록된 맵 / 미등록 맵 / 빈 슬롯 / 패키지 잠금 슬롯)
- 등록된 맵: 유저 평가수 표시 (좋아요/플레이 수)
- 등록 해제 시 초기화 여부 확인 필요
- 추가 규칙대 슬롯은 패키지로 잠금/구매로 해금

**4. 맵 생성**
- 맵 이름 입력 (필수)
- 맵 설명 입력 (필수)
- 맵 배경 선택 (Easy/Normal/Hard 난이도별)
- 샘플 사용하기 선택지 (Sample 1/2/3, 사용하기 버튼)
- 필수 정보 미입력 시 생성하기 버튼 비활성화
- 필수 정보 입력 시 생성하기 가능

**5. 선택 맵 메인 화면**
- 맵 삭제 버튼, 맵 이름, 도움말 버튼, 닫기(X) 버튼
- 맵 이름, 플레이 설정 정보(카메라 거리, 이동 속도)
- 추가 규칙 표시 (규칙 없음/규칙 있음)
- 등록 상태 표시 (미등록/등록됨 + 코드 ABCDE1234 + 복사)
- 평가수 표시 (좋아요/플레이 수)
- 맵 설정 버튼, 등록 테스트 버튼, 편집하기 버튼

**6. 인게임 - 로딩**
- 로딩 대시(로딩 연출), 진입 전환

**7. 인게임 - 플레이**
- 점프 버튼, 슬라이드 버튼, 상단 미션/점수/시간 UI
- 완주 시 미션 체크 → 성공 처리
- 중간 실패 시 실패 처리

**8. 결과 화면**
- Result 화면 (점수, 보상)
- 성공 결과(SUCCESS, 미션 체크리스트 모두 체크)
- 실패 결과(FAIL, 미션 일부 미달성, 확인 시 메인 화면 이동)
- 달려가기만 했을 때(미션 체크리스트 미출력)

### 중점 확인 사항

1. **필수 입력 검증**: 맵 이름·설명 미입력 시 생성하기 버튼 비활성화 동작 → 데이터 입력 검증의 핵심 케이스
2. **등록 상태 분기**: 미등록/등록됨 상태에 따라 화면 UI(평가수, 코드 ABCDE1234)가 달라지므로 상태별 분기 케이스 분리 필수
3. **등록 해제 시 초기화**: 기획서에 "등록 해제되면 초기화되어야 하나?" 코멘트가 있어 해당 분기 케이스 명시 필요 (확정되지 않은 부분은 확인 가능한 동작만 작성)
4. **달리기 모드**: "다음 TF 때" 코멘트가 있으므로 달리기 버튼은 진입 차단 또는 미구현 상태 확인 범위까지만 작성하고 인게임 흐름은 만들기 기준으로만 작성
5. **샘플 사용하기 분기**: 샘플 선택 시 맵 배경이 자동 적용되는 동작과, 직접 입력 시 동작을 분리하여 검증
6. **결과 화면 3분기**: SUCCESS / FAIL / 달려가기만 했을 때 — 미션 체크리스트 출력 여부와 확인 버튼 동작 분기 확인
7. **인게임 케이스 제외 원칙**: 점프/슬라이드 동작은 "인게임"에 해당하므로 버튼 노출·진입 검증까지만 작성, 인게임 플레이 내부 동작은 작성 제외
8. **예외 케이스**: 슬롯 잠금 상태(패키지 미보유), 등록 코드 복사 실패, 네트워크 단절 시 등록/생성 동작 등은 카테고리 말미에 배치
