### 기능 목록

기획서 본문(텍스트)과 UI 플로우, 첨부 이미지(스크린샷)가 모두 제공되지 않아 분석 가능한 기능을 식별할 수 없습니다. 테스트 케이스 제목 "테스트" 외에 검토 가능한 자료가 없으므로, 추측에 의한 기능 도출은 작성 규칙(추측 금지)에 따라 수행하지 않습니다.

확인 가능한 입력:
- 테스트 케이스 제목: "테스트"
- 기획서 텍스트: 없음
- UI 플로우: 없음
- 첨부 이미지(스크린샷/목업): 확인되지 않음
- TC 예시 양식: 없음 (기본 양식 사용 예정)

### 카테고리 구조

```
└─ (분석 가능한 main 카테고리 없음)
```

자료가 제공되지 않아 main / sub1 / sub2 / sub3 계층을 도출할 수 없습니다. 기획서 또는 스크린샷이 제공되면, 가장 큰 기능 단위를 main으로 잡고 진입 화면(sub1) → 영역/탭(sub2) → 유저 상호작용 단위(sub3) 순으로 세분화하여 트리를 구성하겠습니다.

### TC 건수 배분

자료 부재로 기획서 규모를 가늠할 수 없어, 권고치(100건 이상)에 대한 카테고리별 건수 배분도 수립할 수 없습니다. 자료 수신 후 아래 분배 원칙에 맞춰 배분 계획을 수립할 예정입니다.

- Critical: 전체의 10~15% (게임 진행 불가 / 데이터·재화 손실 / 결제 오류 / 핵심 진입 차단에 한정)
- Major: 전체의 40~50% (주요 기능 동작, 화면 전환, 보상 지급/차감, 알림 시스템 등)
- Normal: 전체의 35~50% (효과음, 애니메이션·연출, 텍스트 표기, 보조 UI 등)

### 중점 확인 사항

자료 수신 시 다음 사항을 우선하여 TC 작성에 반영하겠습니다.

1. 기획서 원문(텍스트) 또는 PDF, 스크린샷·UI 목업 이미지의 실제 첨부 여부
2. UI 플로우(진입 경로, 화면 전환 순서)
3. 콘텐츠 해금/잠금 조건, 사전 조건 명세
4. 화면 내 모든 버튼/탭/토글 동작과 뒤로가기 플로우
5. 레드닷/그린닷 등 알림 시스템의 출력·제거 조건
6. 보상 지급/차감, 재화 변동, 결제 관련 동작
7. 효과음·애니메이션·연출 등 부가 요소
8. 예외 상황(앱 종료/재실행, 네트워크 단절, 조건 미달, 오류 케이스) — 각 카테고리 말미 배치
