# TC 설계 계획

## 1. 기능 목록 (기획서에서 파악한 전체 기능)

### A. 콘텐츠 개요
- 쿠키 스토리 던전: 일반 모드(맵 6종), 챌린지 모드(맵 6종)
- 일반 모드: 다이얼로그 + 일반 전투
- 챌린지 모드: 중간 보스 + 보스 전투 (기존 스토리모드 보스 재활용)

### B. 튜토리얼 — EP1 흐름 (다이얼로그 1 + 신규 쿠키 + 체험쿠키 고정)
- 0. 다이얼로그 출력 (AUTO/SKIP/진행/화자명)
- 1. 튜토리얼 가이드 팝업 출력 (스킬 요약 설명)
- 2. 쿠키 정보 버튼 출력 (좌측 상단)
- 2-1. 쿠키 정보 버튼 터치 → 쿠키 스킬 설명 팝업 출력
- 3. 일시정지 기능이 포함된 튜토리얼 진행
- 4. 전투 진행

### C. 튜토리얼 — EP2 흐름 (다이얼로그 2 + 신규 쿠키 + 체험쿠키 고정)
- 0. 다이얼로그 출력
- 1. 쿠키 정보 버튼 출력
- 2. 전투 진행 — 쿠키 특성에 맞춘 특수 전투 (연출 추가) ex) 달빛술사: 특수스킬 쿨타임에 맞춘 적 스폰 후 한 번에 무력화 경험
- 3. 전투 종료 후 다이얼로그 출력

### D. 튜토리얼 — EP3~6 흐름 (다이얼로그 3~6 + 자유 쿠키 + 신쿠키 1개 고정)
- 0. 다이얼로그 출력
- 1. 일반 전투
- 2. 전투 종료 후 다이얼로그 출력

### E. 쿠키 스토리 던전 — 일반 모드
- 시작 UI: 배경 색, 콘텐츠 이름(쿠키 스토리 던전 Ch.A), 진행 중인 에피소드 제목, 에피소드 넘버(Ep.1 형식)
- 게임 진행 화면: 좌측 상단 쿠키 정보 버튼(EP1,2 한정), 제한 시간 없음
- 일시정지 UI: 콘텐츠 이름/에피소드 제목 표시, 목표 표시(휑해서 넣음), 다시하기 없음, 그만하기/계속하기 버튼
- 결과 화면 UI — CLEAR: 최초 클리어 보상, 최초 클리어 시 소탕 기능 해제 정보, 갱신된 소탕 보너스 %값, 나가기 버튼
- 결과 화면 UI — FAIL: "실패 시 보상을 획득할 수 없습니다." 문구, 다시하기/나가기 버튼

### F. 쿠키 스토리 던전 — 챌린지 모드
- 시작 UI: 콘텐츠 이름(챌린지 모드), 현재 단계 넘버(1단계 형식)
- 게임 진행 화면: 일시정지 외 모든 버튼 제거, 쿠키 정보 버튼도 삭제(일시정지로 이동), 제한 시간 없음
- 일시정지 UI: 콘텐츠 이름 및 단계 표시(일반 모드와 동일 구조)
- 결과 화면 UI: 최초 클리어 보상만 표시, 나가기 버튼

  - sub2: 일시정지 기능 안내 튜토리얼 → (sub3) 안내 출력 / 일시정지 동작

  - sub2: FAIL 화면 → (sub3) 출력/보상 없음 문구/다시하기/나가기

### Main 3: 쿠키 스토리 던전 - 챌린지 모드

## 4. 중점 확인 사항

1. **튜토리얼 적용 범위 명확화** — 스킬 요약 설명 팝업, 쿠키 정보 버튼, 쿠키 스킬 설명 팝업은 **EP1, EP2에 한해서**만 노출됨 (EP3~6은 일반 전투). TC 작성 시 적용 에피소드를 명시.

2. **모드별 게임 진행 화면 차이** — 챌린지 모드는 "일시정지 버튼을 제외한 나머지 버튼은 제거", "쿠키 정보 버튼도 삭제(일시정지로 이동)"로 일반 모드와 차이가 있음. 챌린지 모드 TC는 차이점 위주로 작성.

3. **결과 화면 차이** — 일반 모드 CLEAR/FAIL 모두 존재 / 챌린지 모드는 최초 클리어 보상만 표시되고 다시하기 버튼이 없는 것으로 보임 → 화면 구성 차이 TC 분리.

4. **최초 클리어 보상 vs 갱신 소탕 보너스** — 일반 모드 CLEAR 화면에서 최초 클리어 시에만 소탕 기능 해제 정보가 제공된다는 조건부 동작. 최초/재클리어 구분 TC 필요.

5. **소탕 보너스 %값 갱신 표시** — 일반 모드 CLEAR에서 "갱신되는 소탕 보너스 %값 표시" (예: 예상 소탕 보너스 +60%) → 갱신 동작 TC.

6. **다이얼로그 공통 UI** — AUTO, SKIP, 진행 버튼, 화자명 라벨(예: "크림스다맛 쿠키"). EP1·EP2·EP3~6 모두 다이얼로그를 통해 시작/종료되므로 공통 동작 케이스 포함.

7. **제한 시간 없음** — 모드 공통 사양. TC에 반영.

8. **그만하기/계속하기/다시하기/나가기 버튼 동작** — 화면 전환 케이스는 각 카테고리 sub1 마지막에 배치 (규칙 3).

9. **연출/효과음** — Normal 우선순위로 CLEAR 연출, 다이얼로그 출력 효과음, 전투 연출(특히 EP2 특수 연출) 포함.

설계가 완료되었습니다. 2단계 JSON 작성을 진행해도 될지 알려주시면 TC 본문 작성에 들어가겠습니다.
