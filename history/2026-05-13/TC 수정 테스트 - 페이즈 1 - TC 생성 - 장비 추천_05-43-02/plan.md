### 기능 목록

기획서에서 파악한 장비 추천 관련 주요 기능:

1. **쿠키 상세화면 진입점**
   - 쿠키 상세화면 내 "추천 조합" 버튼 노출
   - 미보유 쿠키 상태에서도 추천 조합 버튼 표시

2. **추천 조합 팝업 기본 구조**
   - 좌측 탭 구성: 장비 / 어리벅트(아이템) / 잠재력
   - 진입 시 기본 선택 탭(장비)
   - 우측 X(닫기) 버튼
   - 진입한 쿠키 좌상단 표시

3. **추천 장비 탭 UI/기능**
   - 상단 필터 버튼
   - 현재 필터 적용 상태 표시(쿠키 face icon)
   - 검색 버튼
   - 추천 조합 리스트(상위 3개 조합: 1위/2위/3위)
   - 각 조합 4부위(무기/모자/상의/하의) 슬롯 표시
   - 각 조합 사용률(%) 표시 - 소수점 첫째 자리까지(둘째 자리 반올림)

4. **추천 장비 데이터 기준**
   - 챔피언스(명예의 전당) 레이드 상위 1000명 데이터 기반
   - 24시간 단위 갱신
   - 사용된 특정 부위 장비 종류 3개 미만 시 사용된 장비만 표시
   - 누적 플레이 1000회 미만 시 "기록이 충분하지 않습니다." 안내

5. **조합 따라하기 버튼 동작 분기**
   - 분기①: 추천 장비 모두 보유 + 다른 쿠키 미장착 → 가장 높은 단계 장착 + "조합이 적용되었습니다." 토스트
   - 분기②: 일부 미보유 OR 다른 쿠키 장착 중 → 적용 가능 장비만 장착 + 동일 토스트
   - 분기③: 장착 가능 장비 0개 → "장착할 수 있는 장비가 없습니다." 토스트
   - 분기④: 미보유 쿠키 → 버튼 비활성화

6. **장비 슬롯 상세**
   - 장비 슬롯 선택 시 우측에 장비 상세 정보 표시
   - 선택 전 "선택된 정보 없습니다." 안내
   - 선택된 장비 레벨 표시

7. **팝업 종료**
   - X 버튼 터치 시 팝업 종료 및 쿠키 상세화면 복귀

### 카테고리 구조

```
└─ 쿠키 (main)
   └─ 추천 조합 (sub1)
      └─ 장비 추천 (sub2)
         └─ 조합 따라하기 (sub3)
```

### TC 건수 배분

- **전체 목표**: 1건 (사용자 요청: "TC 1개 케이스로 작성")
- **카테고리별 배분**:
  - 쿠키 > 추천 조합 > 장비 추천 > 조합 따라하기: 1건
- **Priority 배분**:
  - Critical: 0건
  - Major: 1건 (조합 따라하기는 핵심 기능 동작이지만 게임 진행 불가/데이터 손실/결제 오류에 해당하지 않으므로 Major로 분류)
  - Normal: 0건

### 중점 확인 사항

1. **단일 TC 작성 제약**: 사용자가 명시적으로 "TC 1개 케이스로 작성"을 요구했으므로, 장비 추천 기능 중 가장 핵심적이고 대표성 있는 케이스 1건만 작성한다. 여러 분기/예외를 한 TC에 묶지 않고 단일 기대결과 원칙을 유지한다.

2. **단일 기대결과 원칙 준수**: 조합 따라하기 분기가 4가지(①~④)이지만, 가장 핵심 분기인 "추천 장비 모두 보유 + 다른 쿠키 미장착 → 장착 성공 + 토스트 출력" 분기 하나에 집중한다.

3. **카테고리 계층 세분화**: main(쿠키) → sub1(추천 조합) → sub2(장비 추천) → sub3(조합 따라하기) 4단계 모두 채워 작성한다.

4. **문체 규칙 준수**:
   - "~하면" → "~시,"
   - "정상" 표현 금지
   - 수동형 표현 사용

5. **Priority 판단**: 조합 따라하기는 핵심 기능이지만 결함 발생 시 게임 진행 불가나 데이터 손실로 직결되지 않으므로 Major로 분류한다.

6. **구체적 조건 명시**: 사전 조건(추천 장비 모두 보유, 타 쿠키 미장착)과 기대결과(장착 + 토스트)가 한 TC에 명확히 드러나도록 작성한다.
