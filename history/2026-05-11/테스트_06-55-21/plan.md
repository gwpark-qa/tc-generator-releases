### 기능 목록

기획서에서 파악된 주요 기능을 흐름 순서대로 정리합니다.

**[A. 레이드 모드 진입 화면]**
1. 로비 → 햄버거 버튼 → 레이드 모드 진입 (화면 전환)
2. 상단 뒤로가기 버튼
3. 상시/이벤트 토글 탭
4. 등장 보스 정보 카드 (보스명, 등장 보상, 도전 정보, 슈퍼에픽 세트 정보 최대 3종)
5. 잠금 아이콘 터치 시 잠금 안내 팝업
6. 보스 등장 스테이지 배경 이미지 표시
7. 보스 카드 선택 → 레이드 모드 로비 진입 전환

**[B. 레이드 모드 로비]**
1. 싱글/멀티 전환 토글 (Default: 싱글)
2. 토글 활성화 모드 기준으로 화면 진입 시 해당 모드 화면 표시
3. 멀티 모드 진입 시 매칭 화면으로 전환
4. 보스 정보 표시 영역 (보스명, 초상화, 배경)
5. 보스 정보 좌측 팝업 출력
6. 난이도 변경 버튼 (현재 난이도 표시)
7. 난이도 변경 팝업 (단계 단계 표시, 입장 전투력 표시)
8. 보스 변경 (상단 좌우 버튼)
9. 난이도 변경 팝업 중복 클릭 시 닫힘 (토글 동작)
10. 입장 전투력 미달 시 단계 클릭 → 토스트 출력 (equipment.dungeon.not.host)
11. 이전 난이도 미클리어 시 → 토스트 (single_boss_challenge.locked)
12. 잠금 단계 자물쇠 표시
13. 빠른 난이도 변경 버튼
14. 빠른 난이도 변경 비활성화/토스트 처리
15. 보상 정보 영역 표시
16. 보상 상세 정보 팝업 (보스 변경 좌우 버튼, 난이도별 보상 탭)
17. 클리어 기본 보상 / 연관 보상 표시
18. 입장 전투력 / 추천 속성 / 적용 조건 표시
19. Play 버튼
20. 배수 설정 버튼 (기본 최대 3장, 패스 구매 시 최대 5장)
21. Play 버튼 비활성화 + 토스트 (이전 난이도 미클리어 시)
22. 티켓 입출 5장 → 2장 변경
23. 난이도 10단계 표시
24. 일반 레이드 랭킹 진입 버튼 제거 확인

**[C. 레이드 인게임 진입 (전환만)]**
- Play 버튼 터치 → 로딩화면 진입 (인게임 자체는 제외)
- 클리어 화면 보상 표시 / 나가기 → 로비 복귀 (UI 전환 케이스만)
- 트로피/축복/무지개 상자/랜덤 상자 제거 확인 (UI 잔존 검증)
- 다시까기 기능 제거 확인
- 획득 아이템 정보 표시 영역 (12종 최적화, 초과 시 스크롤)
- 아이템 클릭 시 툴팁 출력
- 드랍 연출 약 3초

**[D. 챌린지 레이드 모드 로비]**
1. 챌린지 모드 탭/버튼 → 챌린지 레이드 로비 전환
2. 챌린지 보스 진열 (보스 일러스트, 보스명, 보상 표시)
3. 입장 정보, PLAY 버튼
4. 챔피언 점수 팝업
5. 챕터 보상 미션 아이콘 표시
6. 코코방울 축복/트로피/무지개 상자 영역 제거 확인
7. 4인 멀티 → 2인 멀티 변경 확인
8. 트로피 시스템 제거
9. 클리어 화면 트로피 영역 제거 확인

**[E. 제작소 화면]**
1. 로비 → 햄버거 → 제작소 진입 전환
2. 뒤로가기 버튼
3. 제작소 NPC / 타이틀
4. 메뉴 정렬 (장비 제작 → 장비 연금 → 아티팩트 제작 → 아이템 제작)
5. 장비 연금 메뉴 버튼 추가
6. 장비 연금 진입 화면 전환

**[F. 레이드 모드 탑레벨 화면]**
- UI 변경 자체는 제외 대상이므로, 진입 전환 케이스만 작성
- 햄버거 → 성장 탭 → 레이드 모드 진입 전환
- 보스 리스트 → 진입 버튼(→) → 해당 보스 로비 진입 전환

### TC 건수 배분

전체 예상: **약 75~85건**

기획서 규모상 무리하게 100건을 채우는 것은 불필요해 보입니다. 제외 항목(연금술/탑레벨 UI 변경/데이터 테이블/인게임)이 많아 실제 TC 작성 대상이 제한적입니다.

|---|---|---|---|---|
| 레이드 모드 진입 화면 | 8~10 | 1 | 5 | 3 |
| 레이드 모드 로비 | 30~35 | 4 | 18 | 10 |
| 레이드 클리어 화면 | 10~12 | 1 | 5 | 5 |
| 챌린지 레이드 로비 | 12~15 | 1 | 7 | 6 |
| 제작소 | 6~8 | 1 | 3 | 3 |
| 탑레벨 화면 진입 | 3~5 | 0 | 2 | 2 |
| **합계** | **약 75~85건** | **8~10건 (10~12%)** | **38~42건 (45~48%)** | **28~32건 (35~40%)** |

### 중점 확인 사항

1. **제외 항목 엄격 적용**
   - 인게임 플레이 자체 케이스 작성 금지 (Play 버튼 → 로딩 진입까지만)
   - 연금술 이벤트 삭제 / 장비 연금 상시 시스템 편입 / 연금술 항아리 드랍 관련 케이스 제외
   - 탑레벨 UI 변경(6종 보스 정보 표시 변경)은 화면 전환 케이스만 작성
   - 데이터 테이블, 이펙트 요청 리스트 제외

2. **제거 기능 잔존 여부 검증**
   - 코코방울 연출/축복/축복 포인트
   - 트로피 시스템/트로피 상자/트로피 포인트
   - 무지개 상자 이벤트
   - 랜덤 상자 관련 기능
   - 일반 레이드 랭킹 진입 버튼
   - 다시까기 기능 (티켓 배율로 대체)
   - 클리어 화면 내 트로피/축복/무지개 영역

3. **토스트 메시지 정확한 ID 반영**
   - 입장 전투력 미달: `equipment.dungeon.not.host`
   - 이전 난이도 미클리어: `single_boss_challenge.locked`

4. **수치/스펙 정확 반영**
   - 티켓 입출: 5장 → 2장 변경
   - 1회 플레이 최대 소모 티켓: 기본 3장 / 패스 시 5장
   - 난이도 10단계
   - 멀티 인원: 4인 → 2인
   - 아이템 표시: 최대 12종 + 스크롤
   - 드랍 연출: 약 3초

5. **싱글/멀티 토글 Default(싱글) 상태 검증**

6. **화면 전환 케이스 누락 방지** (각 카테고리 마지막에 배치)
   - 로비 → 레이드 진입 화면
   - 진입 화면 → 레이드 로비
   - 레이드 로비 ↔ 챌린지 로비
   - 로비 → 제작소 → 장비 연금
   - 탑레벨 → 보스 로비

7. **Priority 판단 주의**
   - Critical: Play 진입 차단, 결제(패스 구매 영향) 관련, 재화 차감 오류, 크래시 가능 지점에 한정
   - 단순 화면 전환·표시 기능은 Major 이하로 분류

8. **카테고리 통합 원칙**
   - 챌린지/제작소/탑레벨도 각각 main으로 두되, 진입 버튼 시작점은 해당 main 내 첫 sub1로 시작
   - 화면 이동/예외 케이스는 각 카테고리 마지막에 배치

위 계획대로 진행해도 될지 검토 부탁드립니다. 승인 시 JSON 형식으로 TC를 작성하겠습니다.
