기획서를 분석했습니다. "수면 보호막"은 로그라이크 모드의 공용/스페셜 챕터에서 노래하는 인형 몬스터가 사용하는 인게임 시스템 기믹입니다. 규모가 작은 기획서이므로 100건을 무리하게 채우지 않고 커버리지 100% 기준으로 작성 계획을 수립합니다.

### 기능 목록
1. 보호막 부여 — 노래하는 인형 몬스터(보호막 부여자)가 대상에게 수면 보호막을 부여
2. 보호막 수치 산정 — 보호막 부여자(노래하는 인형)의 최대 체력에 따라 보호막 양 결정
3. 보호막 적용 대상 — 노래하는 인형이 보호막을 사용하는 몬스터
4. 수면 공격에 의한 보호막 즉시 제거 — 수면 공격 피격 시, 보호막이 즉시 사라짐
5. 일반 피격에 의한 보호막 차감/제거 — 수면 공격이 아닌 일반 피격에 의해서도 보호막이 차감되어 제거될 수 있음
6. 보호막 우선 차감 — 보호막 보유 중 피격 시, 본체 체력보다 보호막이 우선 차감
7. 보호막바 UI 표시 — 대상 캐릭터의 체력바와 함께 보호막바가 출력됨
8. 보호막바 색상 — 이펙트에 어울리는 색상(보라/핑크 계열)으로 표시
9. 보호막바 사선 패턴 표시 — 체력바 상에 사선 패턴으로 보호막 영역 표시
10. 보호막 소진 시 UI 제거 — 보호막이 0이 되면 보호막바가 제거됨
11. 보호막 잔여량 시각화 — 피격에 따라 보호막바 잔량이 갱신됨
12. 보호막 부여 이펙트
13. 보호막 제거 이펙트
14. 보호막 관련 사운드
15. 등장 챕터 — 공용 챕터, 스페셜 챕터에 등장
16. 모드 — 로그라이크 모드에서 동작

### TC 건수 배분
기획서 규모가 작아 총 약 18~22건으로 작성 예정 (커버리지 100% 우선, 무리한 100건 채움 지양).

- **Critical (약 2~3건, 12~15%)**
  - 보호막 부여 동작 (핵심 기믹 작동)
  - 수면 공격 피격 시 보호막 즉시 제거 (핵심 메커니즘)
  - 보호막 우선 차감 (체력 보호 핵심 동작)

- **Major (약 9~11건, 45~50%)**
  - 노래하는 인형 등장 시 보호막 부여
  - 보호막 수치 = 부여자 최대 체력 비례 산정
  - 일반 피격에 의한 보호막 차감
  - 일반 피격으로 보호막 소진 시 제거
  - 보호막바 출력/잔량 갱신/소진 시 제거
  - 공용/스페셜 챕터, 로그라이크 모드 적용

- **Normal (약 7~8건, 35~40%)**
  - 보호막바 색상(이펙트 어울리는 색)
  - 사선 패턴 표시
  - 부여/제거 이펙트
  - 사운드 출력
  - 수치 시각 표기

### 중점 확인 사항
1. **기획의도 vs 기능 구분**: "달빛술사의 수면을 더 강조하기 위한 새로운 보호막"은 의도이므로 TC 제외. 단, '수면 피격 시 즉시 사라짐', '일반 피격으로도 사라짐', '부여자 최대 체력 기반 부여' 등은 기능 스펙이므로 TC 작성 대상.
2. **부여자/적용 대상 정확성**: 기획서상 "노래하는 인형"이 보호막을 사용하는 몬스터로 명시 → 부여 주체/대상 혼동 주의. (피격 대상이 누구인지는 텍스트로 단정 불가하므로 추측 금지)
3. **UI 정확 표현**: 스크린샷에서 체력바 우측 끝부분에 사선(빗금) 패턴으로 보호막 영역이 표시되며, 보라/핑크 계열 색상 사용 → "이펙트에 어울리는 색상", "사선 패턴" 위치/형태 명시.
4. **수면 제거 vs 일반 피격 차감 구분**: 수면 공격은 '즉시 제거', 일반 피격은 '차감 후 0이 되면 제거'로 동작 차이 분리 TC.
5. **인게임 직접 플레이 동작 제외 범위 검토**: 본 기믹은 인게임 시스템이나, 기획서 자체가 해당 기믹 스펙을 다루므로 기믹의 동작/UI/연출에 한정해 작성하고, 일반 전투/조작 등 무관한 인게임 케이스는 작성하지 않음.
6. **로그라이크 모드 한정**: '모드' 속성이 로그라이크로 명시 → 다른 모드에서는 등장하지 않음을 검증 케이스로 포함 검토.
7. **등장 챕터 분리**: 공용/스페셜 챕터 모두 등장 → 각각 동작 케이스 분리.
8. **추측 금지 영역**: 보호막 수치의 정확한 비율(예: 최대 체력의 몇 %), 이펙트/사운드의 구체 형태, 부여 트리거 조건 등은 기획서에 명시되지 않음 → 일반화하여 작성하고 구체 수치/문구는 인용하지 않음.
